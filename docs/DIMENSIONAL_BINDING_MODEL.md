# Dimensional Binding Model

## Purpose

This repository does not treat memory, identity, intent, or agency as flat yes/no states.

Binding is resolved across a multi-dimensional support field.

A formation may appear valid in one dimension and still fail binding because another dimension cannot carry it.

## Core position

```text
No memory, identity, intent, or agency formation binds merely because it is fluent, coherent-looking, or internally generated.

Binding requires support across continuity, coherence, standing, replay, contamination posture, integrity, and target legitimacy.
```

## Dimensional field

The public proof surface uses safe surrogate dimensions only:

```text
continuity        Does the formation remain connected to a lawful prior chain?
coherence         Does the formation remain internally stable under current conditions?
standing          Does the formation have authority to bind as this kind of object?
replay basis      Can the same formation posture be reproduced under the same conditions?
integrity         Is the boundary itself intact?
contamination     Has the field been polluted or redirected?
target legitimacy Is the requested bind target allowed at this public layer?
```

## Why flat validation fails

A flat model asks:

```text
Is this memory/identity/intent/agency claim valid?
```

That is insufficient.

A governed boundary asks:

```text
Does this formation have enough dimensional support to bind into the target layer without corrupting continuity, identity, agency posture, or future replay?
```

## Consequence of dimensional failure

The failure mode is not always REFUSE.

Different dimensional failures resolve differently:

```text
continuity failure     → REBOUND
coherence failure      → HALT
standing failure       → REFUSE_SELF
contamination signal   → QUARANTINE
no bind target         → OBSERVE
protected agency bind  → REFUSE_SELF at the public layer
valid memory target    → BIND_MEMORY
valid identity target  → BIND_IDENTITY
```

## Counterfactual support

The counterfactual verifier proves that binding is not decorative.

Starting from a valid bind case, removing required support excludes binding:

```text
continuity removed       → no bind
coherence removed        → no bind
standing removed         → no bind
replay removed           → no bind
contamination introduced → no bind
```

This shows the binding decision is load-bearing across dimensions.

## Public/protected boundary

This document describes the public dimensional surface only.

It does not expose protected identity kernels, memory scoring law, self-reference mechanics, continuity substrate, field equations, agent/council internals, or production runtime internals.

## Clean line

```text
The boundary is not a flat classifier.

It is a dimensional admission surface: a formation binds only when every required support dimension can carry the consequence of that binding.
```
