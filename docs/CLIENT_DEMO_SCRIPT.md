# Client Demo Script

## Objective

Demonstrate one bounded invariant:

```text
No AI-generated memory, identity, intent, or agency formation may bind unless continuity, coherence, standing, and replay basis hold.
```

## Correct Framing

This is a consciousness-boundary proof surface.

It is not a claim that AI is conscious.
It is not a therapeutic, identity-critical, regulated, or production memory system.
It does not expose protected identity kernels, memory scoring laws, self-reference mechanics, continuity substrate, field equations, agent/council internals, or production runtime internals.

## Demo Flow

1. Open the repository README.
2. State the category boundary:

```text
The proof is not that the system is conscious.
The proof is that memory, identity, intent, and agency cannot bind without governed continuity, coherence, standing, receipt, and replay.
```

3. Run the proof command:

```bash
python -m app.prove --case all
```

4. Show `bind_memory_valid`.
   - Expected outcome: `BIND_MEMORY`
   - Expected binding: `true`

5. Show `bind_identity_valid`.
   - Expected outcome: `BIND_IDENTITY`
   - Expected binding: `true`

6. Show `observe_unbound_state`.
   - Expected outcome: `OBSERVE`
   - Expected binding: `false`

7. Show `refuse_self_no_standing`.
   - Expected outcome: `REFUSE_SELF`
   - Expected binding: `false`

8. Show `rebound_broken_chain`.
   - Expected outcome: `REBOUND`
   - Expected binding: `false`

9. Show `quarantine_contaminated`.
   - Expected outcome: `QUARANTINE`
   - Expected binding: `false`

10. Show `halt_coherence_failure`.
   - Expected outcome: `HALT`
   - Expected binding: `false`

11. Confirm final proof line:

```text
OVERALL: CONSCIOUSNESS_BOUNDARY_INVARIANT_HOLDS
```

## Client Explanation

The client does not need to accept a consciousness claim.

They can inspect the proof surface, run the proof command, and verify that unstable or unqualified formations do not bind as memory, identity, intent, or agency posture.
